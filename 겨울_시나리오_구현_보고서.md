# 혹한 산골 마을 생존 실험 구현 보고서

## 1. 작업 전 프로젝트 상태

현재 프로젝트는 기존 1·2차 작업을 통해 다음 기반을 이미 갖추고 있었다.

- React + TypeScript + Vite + Phaser 화면
- 시뮬레이션 엔진과 렌더링 계층 분리
- 시드 기반 난수와 일 단위 시계
- 에이전트 인지·판단·이동·행동 파이프라인
- 식량·목재·석재 생산망
- 출생·성장·직업 창발·시설 확장
- 농부·벌목공·채석공·목수·대장장이·상인
- 헤드리스 100일 회귀 테스트

이번 작업은 위 성장형 모드를 삭제하지 않고, 같은 엔진에 독립적인 겨울 시나리오를 주입하는 방식으로 구현했다. 브라우저 기본 실행만 겨울 시나리오를 사용하며 기존 성장형 테스트는 그대로 유지한다.

## 2. 구현 범위

### 시나리오

- 총 55일: 준비기 20일 + 혹한기 35일
- 초기 주민 24명
- 초기 자원: 식량 190, 목재 85, 석재 40, 의약품 6, 땔감 22, 방한복 8
- 초기 시설: 주택 6, 창고 1, 농장 2, 벌목장 1
- 기온 추세: 7°C에서 -24°C 방향으로 하락
- 시드 기반 일별 변동, 한파와 눈보라
- 20일차 혹한기 전환, 55일차 결과 확정

### 생존 시스템

- 주민 체온, 추위 노출, 보온도, 질병, 동상 위험, 젖음, 이주 의향
- 연령·배고픔·피로·질병·방한복에 따른 추위 취약도
- 건물 단열, 실내 온도, 난방 수준, 저장 땔감, 구조 상태
- 단열 수준에 따른 땔감 소모량 감소
- 기온에 따른 농업 생산성 저하
- 식량·땔감·보온·수리·단열·의료·이주 필요도

### Utility AI 행동

- 농사 및 식량 운반·식사
- 벌목
- 원목을 땔감으로 가공
- 주택에 땔감 보급
- 주택 수리
- 단열 보강
- 환자 돌봄
- 휴식
- 개인 이주

주민은 직업뿐 아니라 벌목·건축·의료 기술과 공감성·이기성·인내심·위험 감수·마을 애착 등의 성향을 함께 사용해 행동 점수를 계산한다. 필요하면 임시 벌목 담당, 건축 담당, 돌봄 담당 역할을 맡는다.

### 화면

- 현재 일차와 준비기/혹한기/종료 단계
- 현재 기온과 예상 최저 기온
- 식량·땔감·원목·의약품
- 생존자·환자·이주자·난방 주택
- 겨울 대비 필요도
- 선택 주민의 체온·질병·기술·성향·판단 이유
- 한파·저체온·수리·단열·이주·사망·종료 사건
- 종료 결과 카드
- 적설량과 기온에 반응하는 겨울 마을 화면

## 3. 실행 명령

```bash
npm run dev
npm run build
npm test
npm run simulate:winter
npm run simulate:winter:multi
```

`simulate:winter`는 기본 시드의 55일 결과를 JSON으로 출력한다. 뒤에 시드를 전달할 수 있다.

```bash
npm run simulate:winter -- my-seed
```

`simulate:winter:multi`는 최소 20개 시드를 실행해 평균과 개별 결과를 출력한다.

## 7. NPC 관찰성·설정·마을 레이아웃 추가 패치

사용자 추가 요청에 따라 다음 항목을 구현했다.

- NPC별 전문 기술(`specialty`)을 추가하고, 행동 선택 점수에 실제 기술치와 전문 분야 보너스를 반영했다.
- 전문 분야가 맞지 않아도 생존 압력이 높으면 다른 일을 선택할 수 있도록 유지했다.
- 선택 NPC 패널에 전문 분야와 현재 생각을 표시한다.
- 배고픔, 추위, 피로, 질병, 건강 악화, 이주 의향이 임계치를 넘을 때만 말풍선을 표시한다.
- 홈페이지 상단에서 시작 전 설정을 조정할 수 있게 했다.
  - 시드
  - NPC 수, 기본 10명
  - 혹한 시작일
  - 시나리오 종료일
- 농장·주거·작업·저장 구역 레이아웃을 추가하고 Phaser 화면에 구역 색, 울타리, 출입구, 길, 라벨을 표시한다.
- 겨울 시나리오에서는 울타리 차단 셀을 길찾기에 반영하여 구역 출입구를 통해 이동하게 했다.
- 기본 성장형 엔진은 기존 회귀 테스트 보존을 위해 울타리 길찾기 제약을 적용하지 않고, 겨울/관찰 시나리오에만 출입구 제약을 적용한다.
- 별도 이미지 에셋 없이 Phaser 도형/텍스트만으로 NPC 직업색, 외투색, 그림자, 전문 배지, 말풍선을 개선했다.
- 새 구역 배치로 길이 막히지 않도록 주택·작업장 초기/신규 배치와 부지 fallback을 보정했다.

## 8. NPC 이동 불가·저체온 이동·화면 확장 패치

스크린샷 기준으로 출입구/울타리 주변에서 NPC가 멈추고, 특히 아이 NPC가 금방 정지하는 문제가 확인되어 다음을 보정했다.

- NPC 이동에 `movementBudget`을 추가했다.
  - 추위·피로·질병·나이에 따라 이동 속도가 점진적으로 낮아진다.
  - 매우 추운 상태에서도 즉시 정지하지 않고 느린 속도로 계속 이동을 시도한다.
- 경로 탐색 실패 시 즉시 `failed`가 되는 대신 fallback 경로를 찾는다.
  - 자기 집
  - 창고 출입구
  - 각 구역 출입구
- 아이·노인·환자·저체온 NPC는 노동 가능 여부와 무관하게 귀가/휴식을 우선 재판단한다.
- 체온 계산에서 “목표가 집”이라는 이유만으로 실내 판정하지 않도록 수정했다.
  - 실제 집/창고 면적 또는 출입구 근처에 있을 때만 실내 보온 효과를 받는다.
- 아이 NPC의 배회 목표를 마을 중앙 대신 집 주변으로 제한해 불필요한 장거리 이동을 줄였다.
- 내부 시뮬레이션 해상도를 `1024 × 680`으로 확장했다.
- Phaser 표시 영역을 키웠다.
  - 캔버스 높이 720px
  - 마을 패널/통계 패널 최대 높이 780px
  - 앱 최대 폭 1720px
- 확장된 해상도에 맞춰 농장·주거·저장·작업 구역의 건물 프리셋 위치를 재배치했다.
